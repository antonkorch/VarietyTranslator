# VarietyTranslator

**VarietyTranslator** — это кроссплатформенное десктопное приложение для управления, сортировки и подготовки к экспорту списков сортов растений (varieties). Приложение разработано на языке **F#** с использованием фреймворка **Avalonia UI** для графического интерфейса и базы данных **SQLite** для хранения данных.

---

## Ключевые возможности

* **Динамическая фильтрация и поиск**: Быстрый поиск сортов по текстовому запросу и фильтрация по компаниям-производителям (Benary, PanAmerican, Floranova и др.) напрямую из SQLite.
* **Интерактивный перенос позиций**: Выбор нескольких элементов из левого (исходного) списка и перенос их в правый (рабочий) список перевода одним кликом.
* **Удобное управление выбором**: Клик по всей площади строки активирует чекбокс и подсвечивает её нежно-зеленым пастельным фоном для лучшей визуальной обратной связи.
* **Клавиатурная сортировка**: Изменение порядка элементов в правом списке с помощью клавиш клавиатуры: вверх и вниз. Логика перемещения защищена от случайного срабатывания при вводе текста в поисковую строку.
* **Экспорт в Excel**: Асинхронная выгрузка полных данных (включая русские переводы названий сортов из БД) в файл `.xlsx` с использованием библиотеки **ClosedXML**.

---

## Архитектура проекта

Приложение разделено на несколько логических слоев для разделения ответственности и упрощения поддержки:

* **`dbLib.fs`** — слой доступа к данным. Содержит чистые F# структуры (`Variety`, `VarietyFull`) и функции для безопасной работы с SQLite (подключение, выполнение параметризованных запросов и автоматическое закрытие ресурсов).
* **`MainWindow.axaml`** — разметка пользовательского интерфейса (XAML). Содержит стили окон, структуры контейнеров для списков и кнопки управления.
* **`MainWindow.axaml.fs`** — логика представления (Code-behind). Инициализирует элементы управления, подписывается на события интерфейса и клавиатуры, а также управляет обновлением списков.
* **`App.axaml` / `App.axaml.fs`** — точка входа в приложение и хранилище общих стилей.

## Системные требования и зависимости

### Системные требования
* **.NET SDK** (версия 8.0 или 10.0)
* Установленная среда разработки (VS Code + Ionide, JetBrains Rider или Visual Studio)

### NuGet-пакеты
Проект использует следующие внешние зависимости:
* **Avalonia** (версия 11.0+) — графический движок.
* **Microsoft.Data.Sqlite** — библиотека для работы с базой данных SQLite.
* **ClosedXML** — библиотека для генерации отчетов в формате Excel.

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone <ссылка_на_ваш_репозиторий>
cd VarietyTranslator
```

### 2. Установка зависимостей
Установите необходимые пакеты в проект:
```bash
dotnet add package Microsoft.Data.Sqlite
dotnet add package ClosedXML
```

### 3. Подготовка базы данных
Убедитесь, что файл базы данных SQLite `variety.db` расположен по пути `resources/variety.db` относительно корня вашего проекта.

В файле проекта **`VarietyTranslator.fsproj`** внутри тега `<ItemGroup>` обязательно должно присутствовать правило для копирования базы данных в выходную папку сборки:

```xml
<ItemGroup>
  <!-- Ваши файлы .fs -->
  <Compile Include="dbLib.fs" />
  <Compile Include="MainWindow.axaml.fs" />
  <Compile Include="App.axaml.fs" />
  <Compile Include="Program.fs" />
  
  <!-- Копирование базы данных -->
  <None Update="resources/variety.db">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </None>
</ItemGroup>
```

### 4. Сборка и запуск приложения
Выполните сборку и запуск проекта из корневой папки:
```bash
dotnet build
dotnet run
```

## Инструкция по использованию

1. **Поиск сортов**:
    * Выберите компанию в выпадающем списке (или оставьте вариант «Все»).
    * Введите часть названия в поисковую строку и нажмите клавишу **Enter** (или кликните на иконку лупы). Результаты отобразятся на левой панели.
2. **Перенос в список для перевода**:
    * Кликните по нужным строкам в левом списке (они выделятся галочками и подсветятся зеленым цветом).
    * Нажмите кнопку **«Добавить»** на панели инструментов. Элементы скопируются в правую панель, а галочки слева автоматически сбросятся.
3. **Сортировка списка**:
    * Кликните на любой перенесенный элемент справа.
    * Нажимайте клавишу вверх на клавиатуре для перемещения выбранных позиций вверх по списку или вниз — для перемещения вниз.
4. **Удаление элементов**:
    * Выделите ненужные строки справа галочками и нажмите кнопку **«Удалить»** (красная иконка корзины). Список обновится с сохранением правильной сквозной нумерации.
5. **Экспорт отчета**:
    * Нажмите кнопку **«Создать Excel»** (зеленая буква X).
    * Выберите путь и введите имя для нового файла во всплывающем системном окне сохранения.
    * Приложение автоматически сделает запрос в БД по идентификаторам выбранных сортов, сопоставит их с русскими переводами и сохранит готовый файл Excel `.xlsx`.
